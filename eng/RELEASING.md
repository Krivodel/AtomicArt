# Выпуск Atomic Art

## Однократная подготовка подписи

Самоподписанный сертификат даёт проверяемую подпись содержимого, но не подтверждает личность издателя через доверенный удостоверяющий центр. Поэтому на чужом компьютере Windows продолжит показывать неизвестного издателя. Для проверки происхождения нужно отдельно опубликовать открытый сертификат и его отпечаток по доверенному каналу.

Создать сертификат в личном хранилище текущего пользователя и экспортировать только открытую часть:

```powershell
.\eng\New-AtomicArtSigningCertificate.ps1 `
    -PublicCertificatePath 'D:\AtomicArt-Release-Signing.cer'
```

Закрытый ключ нельзя добавлять в Git. Его нужно отдельно сохранить в защищённой резервной копии. Для всех выпусков должен использоваться один и тот же ключ: новый самоподписанный сертификат с тем же именем не является тем же издателем.

## Профили публикации Rider

В меню `Publish To` доступны два профиля:

- `WindowsLocal` выполняет прежнюю обычную публикацию в папку `src\AtomicArt.Desktop\bin\Release\net9.0\publish` без Velopack и GitHub.
- `WindowsGitHub` публикует приложение, создаёт подписанный выпуск Velopack, собирает установщик с выбором папки и загружает выпуск в GitHub Releases.

Для `WindowsGitHub` до запуска Rider должны быть заданы переменные окружения:

```powershell
[Environment]::SetEnvironmentVariable(
    'ATOMICART_CERTIFICATE_THUMBPRINT',
    '0123456789ABCDEF0123456789ABCDEF01234567',
    'User')
[Environment]::SetEnvironmentVariable(
    'GITHUB_TOKEN',
    '...',
    'User')
```

После изменения пользовательских переменных окружения Rider нужно перезапустить. Версия выпуска берётся из свойства `<Version>` файла `AtomicArt.Desktop.csproj`.

Приложение зависит от установленной среды .NET 9 Desktop Runtime.

Оболочка установщика написана на C# и собирается через .NET SDK для .NET Framework 4.8. Комплект C++, Inno Setup и NSIS не требуются. .NET Framework 4.8 входит в поддерживаемые версии Windows 10 и Windows 11.

## Локальная сборка выпуска Velopack

Версия обязана иметь вид `x.x.x`. Без ключа `-PublishToGitHub` сценарий только публикует зависимый от .NET единый файл, создаёт локальные установщик и пакеты Velopack, подписывает приложение и установщик сертификатом из `Cert:\CurrentUser\My`, а затем проверяет подписи и отпечаток. Внутренний установщик Velopack настроен на установку .NET 9 Desktop Runtime при её отсутствии.

Каталог `publish` является промежуточным и может содержать неподписанный `AtomicArt.exe`. Velopack копирует исполняемые файлы в полный пакет, подписывает их вместе с установщиком и не изменяет исходные файлы в промежуточном каталоге. Сценарий проверяет подписи всех `.exe` внутри текущего полного пакета и установщика.

Публичный `AtomicArt-win-Setup.exe` является оболочкой на C# над подписанным установщиком Velopack. Она по умолчанию предлагает `C:\Program Files\Atomic Art`, позволяет выбрать другую папку и запрещает устанавливать приложение в каталог сохранений `%LocalAppData%\AtomicArt` либо в пересекающийся с ним каталог. После выбора пути оболочка запрашивает повышение прав Windows и запускает встроенный Velopack в тихом режиме.

При установке и при применении обновления в защищённый каталог Windows показывает запрос UAC. Сами обновления по-прежнему загружаются приложением через Velopack; повышение прав требуется только для замены файлов в защищённой папке.

```powershell
.\eng\Publish-AtomicArtRelease.ps1 `
    -Version '1.2.3' `
    -CertificateThumbprint '0123456789ABCDEF0123456789ABCDEF01234567' `
    -ReleaseNotesPath '.\release-notes.md'
```

Результат сохраняется в `Releases`. Каталог нужно сохранять между выпусками, чтобы Velopack мог строить разностные обновления.

Если полный пакет указанной версии уже существует, сценарий проверяет его подписи и переиспользует без повторной упаковки. Благодаря этому после неудачной загрузки можно снова запустить профиль `WindowsGitHub`. Для сборки изменённого приложения нужно сначала повысить `<Version>`: содержимое уже созданной версии считается неизменяемым.

## Явная публикация в GitHub Releases

Автоматическая загрузка выполняется только при явной передаче ключа `-PublishToGitHub`. Токен передаётся только через переменную окружения и не записывается в репозиторий:

```powershell
$env:GITHUB_TOKEN = '...'
.\eng\Publish-AtomicArtRelease.ps1 `
    -Version '1.2.3' `
    -CertificateThumbprint '0123456789ABCDEF0123456789ABCDEF01234567' `
    -ReleaseNotesPath '.\release-notes.md' `
    -PublishToGitHub
Remove-Item Env:\GITHUB_TOKEN
```

Приложение использует только стабильные выпуски репозитория `Krivodel/AtomicArt`. Предварительные выпуски в автоматическую проверку не входят.

## Проверка пользователем

Подпись файла можно проверить без установки:

```powershell
$signature = Get-AuthenticodeSignature '.\AtomicArt-win-Setup.exe'
$signature.SignerCertificate.Thumbprint
$signature.Status
```

Для самоподписанного сертификата состояние может указывать на недоверенную цепочку. Это ожидаемо для неизвестного издателя. Недопустимы отсутствие подписи, `HashMismatch` и отпечаток, который отличается от опубликованного.
